页面的重绘

如何在线上直接跳过一个验证方法。

1.重写函数方法即可。

#点击元素以为的位置，隐藏该元素。
$(‘.protocoll-content .layer-main’).click(function(){
event.stopPropagation();
});
$(document).click(function(){
$(‘.protocoll-content .layer-main’).hide();
});

#重新绘制元素，需要注意调用方式，不能直接用该元素绑定事件，由于页面部分重绘导致，找不到该元素。
例如：var pprotocollContent = $(“.layer-main .layer-main-content”).html();
pprotocollContent = pprotocollContent.replace(“@card@”,cardNumber).replace(“@date@”,month+”/“+year);
$(“.layer-main .layer-main-content”).html(pprotocollCont);
//不能直接用一下方式绑定：
$(‘.protocoll-content .layer-main .layer-main-content’).click(function(){
console.log(1)
});
//用以下方式绑定：
$(document).on(‘click’,’.protocoll-content .layer-main .layer-main-content’,function(){
$(‘.protocoll-content .layer-main’).hide();
});

#数据库连接池自动断开
java.sql.SQLException: Couldn’t perform the operation setAutoCommit: You can’t perform any operations on this connection. It has been automatically closed by Proxool for some reason (see logs).
解决方法：错误的意思是说，由于某些原因连接被Proxool自动关闭，出现这种情况，一般有如下原因：

1.可能为连接数据库超时，比如调试时，对一个窗口停留太久。（对Lucene建立的站内搜索时，俺就出现了这个错误~）

2.网上的说法：

<proxool.minimumConnectionCount>5</proxool.minimumConnectionCount>//最小连接数
<proxool.maximumConnectionCount>150</proxool.maximumConnectionCount>//最大连接数
 <proxool.simultaneousBuildThrottle>20</proxool.simultaneousBuildThrottle>//同时最大连接数

连接数够用，但是同时最大连接数设置的小了点，取不到连接导致报错。
