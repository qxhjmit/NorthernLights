javaScript高性能

发表于 2019-06-06

时间:2019-05-28

##数据存储
1.访问字面量和局部变量速度最快,反之访问数组元素和对象成员变量相对较慢.

2.局部变量存在于作用链的起始位置,变量在作用链越深,访问的数据越慢,全局变量位于作用链的最末端.

3.把常用的变量存在局部变量中改善性能.

##DOM编程(重排和重绘)
1.最小化访问DOM,把需要改变的在js中处理完成,一次改变为最优.

2.多次用到DOM节点,存储在变量中在引用.

3.引用HTML集合最好拷贝到一个数组中,由于它连接底层文档,不要直接改变它.

4.离线操作DOM树,使用缓存,减少访问布局的次数

5.动画中绝对定位,使用拖放代理

6.事件委托减少处理器的数量

算法和流程控制

1.改善循环的最佳方式是减少每次迭代的运算量和减少循环迭代次数.

##字符串和正则表达式
1.当连接数量巨大或者尺寸巨大的字符串时,数组合并是唯一在IE及更早版本中性能优化的合理方式.

##快速响应的用户界面
1.任何javaScript任务都不应执行超过100毫秒,过长的时间会导致UI更新出现延迟,产生负面影响.

2.worker在游览器UI线程之外运行代码.

3.可用定时器切割成多个线程.

##Ajax
1.五中常用的数据请求方式:XMLHttpRequest(XHR),动态脚本注入,iframe,comet,multipart XHR

2.XHR发送数据到服务器时,Get方式会更快.Get请求只发送一个数据包,而Post至少发送两个数据包,一个装载头,另一个装载Post正文

3.以即时通信为代表的web应用程序对数据的Low Latency要求，传统的基于轮询的方式已经无法满足，而且也会带来不好的用户体验。于是一种基于http长连接的“服务器推”技术便被hack出来。这种技术被命名为 Comet
