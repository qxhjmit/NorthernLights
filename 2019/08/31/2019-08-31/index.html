title:vue跳转页面出现固定滚动高度。

第一 ,今天遇到一个问题，vue路由跳转的时候，每次刚渲染完页面就出现一个固定的滚动高度，好气，我不想要这样的效果，测试又说这是一个bug，所以我一定是要解决的，由于这个页面是我copy其他人页面来做简单修改的，所以代码并不是我自己写的，所以我首先要找到是否有在那个地方设置了scroll，然后我找了很久代码里面并没有这个功能，我有想了想是不是动态设置了高度或者一些定位元素影响到了，马上我有去看了一下并没有，我气是什么影响的，找不到原因项目又急，所以我就简单的做了一下处理，但是还是没有效果，处理过程如下：
1.我所以是在mouted里面设置了滚动的距离，在这个生命周期里面我设置了滚动的距离为0，在本地测试ok，很高兴的我发布到了线上，结果测试说还是一样，哎哎哎，没办法我只能继续找其他处理方式。
2.有与上述的方式不行，所以我这次想了想，既然在这个生命周期里不行，是不是因为在数据渲染的时候出现了，滚动距离的改变，所以这次我就在你渲染完成之后做处理，我还不信，你不行，所以这次我在updated里面做出里，本地测试ok，线上又不行，game over。
3.上述两种情况又不行，所以没办法，我又想了一下，是不是因为什么因素导致了这个问题，这样都不行，那我就给你设置个定时器，我不信你还不行，然后我就去弄了一个定时器，本地ok，上线又gg，我真的是气死了。
4.这次没办法了，我感觉是我的思路有问题，是不是这些代码无效或者标签在微信上无效，所以这次给他设置了一个alert但是有执行到这个步骤，在测试环境里面也可以执行这个方法，所以又排除这个原因。
5.自己的思路有问题，我就问问其他人，参考一下他们的已意见，结果听了一下别人的思路，突然就想通了，应该不是其他问题，是代码布局的问题，所以我就把么有的步伐一一去掉，用排除法，我ka，我终于找到了，出现问题的原因了，原来是代码多了一段这行代码：

.bg {
height: 20rem;
background: url(../../common/image/pic_没有数据@2x.png) no-repeat;
background-size: 100%;
}
最后终于解决了，这次的文章是为了给我以后查找问题给出思路，有的时候就是这么简单的问题，静下心来，就可以很快的找到他。
