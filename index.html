javaScript高性能

发表于 2019-06-06

时间:2019-05-28

##数据存储
1.访问字面量和局部变量速度最快,反之访问数组元素和对象成员变量相对较慢.

2.局部变量存在于作用链的起始位置,变量在作用链越深,访问的数据越慢,全局变量位于作用链的最末端.

3.把常用的变量存在局部变量中改善性能.

##DOM编程(重排和重绘)
1.最小化访问DOM,把需要改变的在js中处理完成,一次改变为最优.

2.多次用到DOM节点,存储在变量中在引用.

3.引用HTML集合最好拷贝到一个数组中,由于它连接底层文档,不要直接改变它.

4.离线操作DOM树,使用缓存,减少访问布局的次数

5.动画中绝对定位,使用拖放代理

6.事件委托减少处理器的数量

算法和流程控制

1.改善循环的最佳方式是减少每次迭代的运算量和减少循环迭代次数.

##字符串和正则表达式
1.当连接数量巨大或者尺寸巨大的字符串时,数组合并是唯一在IE及更早版本中性能优化的合理方式.

##快速响应的用户界面
1.任何javaScript任务都不应执行超过100毫秒,过长的时间会导致UI更新出现延迟,产生负面影响.

2.worker在游览器UI线程之外运行代码.

3.可用定时器切割成多个线程.

##Ajax
1.五中常用的数据请求方式:XMLHttpRequest(XHR),动态脚本注入,iframe,comet,multipart XHR

2.XHR发送数据到服务器时,Get方式会更快.Get请求只发送一个数据包,而Post至少发送两个数据包,一个装载头,另一个装载Post正文

3.以即时通信为代表的web应用程序对数据的Low Latency要求，传统的基于轮询的方式已经无法满足，而且也会带来不好的用户体验。于是一种基于http长连接的“服务器推”技术便被hack出来。这种技术被命名为 Comet

你不知道的javaScript--中卷

发表于 2019-06-05

##值：
1.javaScript中的数组是通过数字索引的一组任意类型的值。字符串和数组类似，但是他们的行为特征不同，在将字符作为数组来处理是需要特别小心。javaScript中的数字包括”整数”和”浮点型”。
2.null类型只有一个值null，undefined类型也只有一个值undefined。所有变量在赋值之前默认值都是undefined。void运算符返回undefined。
3.数字类型有几个特殊值，包括NaN（意思是“not a number”），更确切地说是“invalid number”、+Infinity，-Infinity和-0。
4.NaN是一个特殊值，他和自身不相等，是唯一一个非自反（自反，reflexive，即x===x不成立）的值，而NaN != NaN为true。
5.一个有意思的事，JSON.stringify(-0)返回为”0”,而JSON.stringify(“-0”)为0。
6.简单标量基本类型值（字符串和数字等）通过值复制来复制/传递，而复合值（对象等），通过引用复制来赋值/传递。javaScript中的引用和其他语言中的引用/指针不同，它们不能只想别的变量/引用，只能指向值。

你不知道的javaScript--中卷

发表于 2019-06-05

##类型：
1.javaScript有七种内置类型：null，undefined，boolean，number，string，object和symbol（符号为ES6新增），当然你可以用typeof查看他们的类型。
2.变量没有类型，但是他们持有的值有类型，类型定义了值的行为特征。
3.我们经常把undefined和undeclared混为一谈，但在javaScript中它们是两码事。undefined是值的一种，undeclared则表示变量还没有被声明过。
4.javaScript却将它们混为一谈，在我们试图访问”undeclared”变量时这样报错：ReferenceError：a is not defined，并且typeof 对undefined和undeclared变量都返回”undefined”,
然而，通过typeof的安全防范机制（阻止报错）来检查undeclared变量，有时是个不错的办法。

listagg的拼接为空

发表于 2019-05-31

时间：2019-06-05
开发时遇到的问题:
1.用 listagg(i.eti_lang_code,’;’)within group(order by eti_lang_code) as eti_lang_code 转化查询为空.
由于eti_lang_code为varchar类型,所以导致拼接为空而number则没有这个问题.
解决方法:加上to_char(),如下:
listagg(to_char(i.eti_lang_code),’;’)within group(order by eti_lang_code) as eti_lang_code

一、多行注释快捷键

　　1.选中你要加注释的区域，用 Ctrl+Shift+C 或者 Ctrl+/ 会加上 // 注释，再重复按一下就会去掉 // 注释。（.js文件中只有 Ctrl+Shift+C 管用，.java文件中都管用）

　　2.选中你要加注释的区域，用 Ctrl+shit+/  会加上 /*...*/ 注释，再用 Ctrl+shit+\  会去掉 /*...*/ 注释。（注意：正斜杠是加注释，反斜杠是去掉注释）

　　3.要修改在eclispe中的命令的快捷键方式我们只需进入 Windows -> Preferences -> General -> keys 设置就行。

二、Eclipse中添加文档注释快捷键

　　1.例如默认的文档注释：

　　　　/**   
　　　　* @author xxx
　　　　*

　　　　*/

　　　　例如：

object.prototype原型

发表于 2019-05-26

时间:2019-05-24

1.访问对象并不存在一个属性，get的操作就会找到对象内部，关联的对象，这个关系实际上就是原型链。
2.所有普通的对象多有内置的object.prototype，指向原型链的顶端，其中tostring()，valueOf()和其他一些通用的功能存在于object.prototype,所有对象多可以引用。
3.使用new调用函数时会把新对象.prototype属性关联到”其他对象上”，带new被称为”构造函数”。
4.虽然javascript有类似其他语言的类功能，但是它实际上是并不能进行复制，只是通过prototype链进行对象内部之间的关联，所以我们不能用继续进行描述，而是用委托更合适，因为对象之间的关系不是复制而是委托。
