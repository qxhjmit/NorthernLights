第二.介绍我对这个vuex的理解。我主要是找了一遍文章很不错写得很详细，但是有一些我要补充一下:
1.当然首先要去了解ES6的语法。
2.state => 基本数据
getters => 从基本数据派生的数据
mutations => 提交更改数据的方法，同步！
actions => 像一个装饰器，包裹mutations，使之可以异步。
modules => 模块化Vuex
简而言之就是上面这个意思，而mapState，mapGetters,mapMutations,mapActions其实就是它们的重命名一样。与一个使用的例子
…mapState({
counts:(state) => state.count
})
…mapMutations({
addnum:’SET_COUMT_ADD’,
reducenum:’SET_COUMT_REDUCE’
})
首先你要知道…的语法是什么作用，其实他相当于一个展开式，展开成一个一个可以使用的数据，我的理解就是一个个参数或者方法，然后你可以在｛｝里面使用里面使用这个属性，值是直接使用参数把对象传递进counts:(state) => state.count而方法是直接使用对象名addnum:’SET_COUMT_ADD’,用’’括起来
当然你如果想自己提交，在mutations可以使用this.$store.commit，在actions中使用this.$store.dispatch，最好使用后一种，因为在 Vuex 中，mutations里面的方法 都是同步事务，意思就是说：比如这里的一个this.$store.commit(‘newNum’,sum)方法,两个组件里用执行得到的值，每次都是一样的，这样肯定不是理想的需求

title:你不知道的相等==/===

第一.相等的两种表达式：
1.==检查值相等，且它是允许类型转换情况下的值的相等性，
2.===检查类型和值相等，不允许类型转换
3.===经常被称为严格相等
第二.一个有趣的现象，很不错记录下来给大家看一下：
var a = [1,2,3]
var b = [1,2,3]
var c = ‘1,2,3’
a == c //true
b == c //true
a == b //false

ES5比较两个值是否相等，只有两个运算符：相等运算符（==）和严格相等运算符（===）。它们都有缺点，前者会自动转换数据类型，后者的NaN不等于自身，以及+0等于-0。JavaScript缺乏一种运算，在所有环境中，只要两个值是一样的，它们就应该相等。
+0 === -0 //true
NaN === NaN // false
Object.is(+0, -0) // false
Object.is(NaN, NaN) // true

一、多行注释快捷键

　　1.选中你要加注释的区域，用 Ctrl+Shift+C 或者 Ctrl+/ 会加上 // 注释，再重复按一下就会去掉 // 注释。（.js文件中只有 Ctrl+Shift+C 管用，.java文件中都管用）

　　2.选中你要加注释的区域，用 Ctrl+shit+/  会加上 /*...*/ 注释，再用 Ctrl+shit+\  会去掉 /*...*/ 注释。（注意：正斜杠是加注释，反斜杠是去掉注释）

　　3.要修改在eclispe中的命令的快捷键方式我们只需进入 Windows -> Preferences -> General -> keys 设置就行。
