在vue项目中向上走马灯，延迟失效

发表于 2019-08-18

1.为了实现广告效果的走马灯，自己参照网上谢了一个，向上翻滚的效果，原本在一个单独的页面上是没有问题的，
效果非常理想，但是一把样式和定时器放进另外一个页面中，就突然出现，延迟效果无效了，为了弄清楚为什么会导致这个
问题，我把各个模块的代码注释掉然后查找是哪个模块影响到了，结果发现是v-for生成的列表好像对这个有影响，至于
为什么会影响到走马灯哪一款，我找了半天不知道，为什么，猜测应该是生命周期生成数据的时候影响到了，但是这个效果
我必须要实现，后来，没办法我就把他独立成一个模块，结果问题就解决了，但是在解决的时候还是遇到了几个坑，我这里
列出来防止以后自己还犯这个错误：
1.在使用模块接收数据的时候，由于父组件数据是异步的，之前没有使用v-if判断数据是否存在，导致在子组件一直拿不到数据。
2.在定时器里的this的指向，没有在外部定义或者用箭头函数去表示，结果在定时器里面是用了this，导致指向问题，一直
拿不到数据，还以为是数据没有传过来，但是我发在create里面可以接收到。最后才查到是这个问题，平时我一定会注意到，
但是查问题差乱了，自己这个细节多没注意到，这是恨不应该的。
3.最后解决了问题，然后又在销毁组件之前，把定时器清除了，这个是很重要的一定要清除定时器，不然他可能没有自己销毁一直
在。

var Ajax={
get: function(url, fn) {
// XMLHttpRequest对象用于在后台与服务器交换数据
var xhr = new XMLHttpRequest();
xhr.open(‘GET’, url, true);
xhr.onreadystatechange = function() {
// readyState == 4说明请求已完成
if (xhr.readyState == 4 && xhr.status == 200) {
// 从服务器获得数据
fn.call(this, xhr.responseText);
}
};
xhr.send();
}
}

var Axios = {
get: function(url) {
return new Promise((resolve, reject) => {
var xhr = new XMLHttpRequest();
xhr.open(‘GET’, url, true);
xhr.onreadystatechange = function() {
// readyState == 4说明请求已完成
if (xhr.readyState == 4 && xhr.status == 200) {
// 从服务器获得数据
resolve(xhr.responseText)
}
};
xhr.send();
})
},
}

co的源码

发表于 2019-06-30

function co(gen) {
var ctx = this;

return new Promise(function(resolve, reject) {
if (typeof gen === ‘function’) gen = gen.call(ctx);
if (!gen || typeof gen.next !== ‘function’) return resolve(gen);

onFulfilled();
function onFulfilled(res) {
  var ret;
  try {
    ret = gen.next(res);
  } catch (e) {
    return reject(e);
  }
  next(ret);
}

});
}
1.首先，co函数接受Generator函数作为参数，返回一个 Promise 对象。

2.在返回的Promise对象里面，co先检查参数gen是否为Generator函数。如果是，就执行该函数，得到一个内部指针对象；如果不是就返回，并将Promise对象的状态改为resolved。

3.接着，co将Generator函数的内部指针对象的next方法，包装成onFulfilled函数。这主要是为了能够捕捉抛出的错误。

4.最后，就是关键的next函数，它会反复调用自身。

function next(ret) {
if (ret.done) return resolve(ret.value);
var value = toPromise.call(ctx, ret.value);
if (value && isPromise(value)) return value.then(onFulfilled, onRejected);
return onRejected(new TypeError(‘You may only yield a function, promise, generator, array, or object, ‘

+ 'but the following object was passed: "' + String(ret.value) + '"'));

}

上面代码中，next 函数的内部代码，一共只有四行命令。

1.第一行，检查当前是否为 Generator 函数的最后一步，如果是就返回。

2.第二行，确保每一步的返回值，是 Promise 对象。

3.第三行，使用 then 方法，为返回值加上回调函数，然后通过 onFulfilled 函数再次调用 next 函数。

4.第四行，在参数不符合要求的情况下（参数非 Thunk 函数和 Promise 对象），将 Promise 对象的状态改为 rejected，从而终止执行。

function* fibonacci() {
let [prev, curr] = [0, 1];
for (;;) {
[prev, curr] = [curr, prev + curr];
