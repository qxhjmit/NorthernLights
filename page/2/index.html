斐波那契数列

发表于 2019-08-27

1.何为斐波那契数列，例如 1,2,3,5,8,13,21,34……
function* fibonacci() {
let [prev, curr] = [0, 1];
for (;;) {
[prev, curr] = [curr, prev + curr];
yield curr;
}
}

for (let n of fibonacci()) {
if (n > 1000) break;
console.log(n);
}

co的源码

发表于 2019-08-27

function co(gen) {
var ctx = this;

return new Promise(function(resolve, reject) {
if (typeof gen === ‘function’) gen = gen.call(ctx);
if (!gen || typeof gen.next !== ‘function’) return resolve(gen);

onFulfilled();
function onFulfilled(res) {
  var ret;
  try {
    ret = gen.next(res);
  } catch (e) {
    return reject(e);
  }
  next(ret);
}

});
}
1.首先，co函数接受Generator函数作为参数，返回一个 Promise 对象。

2.在返回的Promise对象里面，co先检查参数gen是否为Generator函数。如果是，就执行该函数，得到一个内部指针对象；如果不是就返回，并将Promise对象的状态改为resolved。

3.接着，co将Generator函数的内部指针对象的next方法，包装成onFulfilled函数。这主要是为了能够捕捉抛出的错误。

4.最后，就是关键的next函数，它会反复调用自身。

function next(ret) {
if (ret.done) return resolve(ret.value);
var value = toPromise.call(ctx, ret.value);
if (value && isPromise(value)) return value.then(onFulfilled, onRejected);
return onRejected(new TypeError(‘You may only yield a function, promise, generator, array, or object, ‘

+ 'but the following object was passed: "' + String(ret.value) + '"'));

}

上面代码中，next 函数的内部代码，一共只有四行命令。

1.第一行，检查当前是否为 Generator 函数的最后一步，如果是就返回。

2.第二行，确保每一步的返回值，是 Promise 对象。

3.第三行，使用 then 方法，为返回值加上回调函数，然后通过 onFulfilled 函数再次调用 next 函数。

4.第四行，在参数不符合要求的情况下（参数非 Thunk 函数和 Promise 对象），将 Promise 对象的状态改为 rejected，从而终止执行。

你不知道的javaScript--中卷

发表于 2019-08-27

##值：
1.javaScript中的数组是通过数字索引的一组任意类型的值。字符串和数组类似，但是他们的行为特征不同，在将字符作为数组来处理是需要特别小心。javaScript中的数字包括”整数”和”浮点型”。
2.null类型只有一个值null，undefined类型也只有一个值undefined。所有变量在赋值之前默认值都是undefined。void运算符返回undefined。
3.数字类型有几个特殊值，包括NaN（意思是“not a number”），更确切地说是“invalid number”、+Infinity，-Infinity和-0。
4.NaN是一个特殊值，他和自身不相等，是唯一一个非自反（自反，reflexive，即x===x不成立）的值，而NaN != NaN为true。
5.一个有意思的事，JSON.stringify(-0)返回为”0”,而JSON.stringify(“-0”)为0。
6.简单标量基本类型值（字符串和数字等）通过值复制来复制/传递，而复合值（对象等），通过引用复制来赋值/传递。javaScript中的引用和其他语言中的引用/指针不同，它们不能只想别的变量/引用，只能指向值。

1.为什么要写一个深拷贝，因为这个平时的时候用到的地方很多，当然主要的是面试的时候被问到，知道原理但是不怎么会写，这当然不能允许了，
所以，今天我来写一个深拷贝，打扰代码可以网上找啦，主要还是讲下什么是深拷贝，这个涉及到堆和栈。
当然深拷贝主要是针对的引用类型。说到引用类型，就不得不说一下什么是基本类型和引用类型了。
基本类型：number，string，boolean，null，undefined，symbol以及未来ES10新增的BigInt(任意精度整数)七类。
引用类型：(Object类)有常规名值对的无序对象{a:1}，数组[1,2,3]，以及函数等。
基本数据类型只会在栈内存会新开辟一个内存。
而引用数据类型–名存在栈内存中，值存在于堆内存中，但是栈内存会提供一个引用的地址指向堆内存中的值
当b=a进行拷贝时，其实复制的是a的引用地址，而并非堆里面的值。
而当我们a[0]=1时进行数组修改时，由于a与b指向的是同一个地址，所以自然b也受了影响，这就是所谓的浅拷贝了。
那，要是在堆内存中也开辟一个新的内存专门为b存放值，就像基本类型那样，岂不就达到深拷贝的效果了，所以就有了上述3种方式进行深拷贝啦。

你不知道的javaScript--中卷

发表于 2019-08-27

##类型：
1.javaScript有七种内置类型：null，undefined，boolean，number，string，object和symbol（符号为ES6新增），当然你可以用typeof查看他们的类型。
2.变量没有类型，但是他们持有的值有类型，类型定义了值的行为特征。
3.我们经常把undefined和undeclared混为一谈，但在javaScript中它们是两码事。undefined是值的一种，undeclared则表示变量还没有被声明过。
4.javaScript却将它们混为一谈，在我们试图访问”undeclared”变量时这样报错：ReferenceError：a is not defined，并且typeof 对undefined和undeclared变量都返回”undefined”,
然而，通过typeof的安全防范机制（阻止报错）来检查undeclared变量，有时是个不错的办法。

javaScript高性能

发表于 2019-08-27

时间:2019-05-28

##数据存储
1.访问字面量和局部变量速度最快,反之访问数组元素和对象成员变量相对较慢.

2.局部变量存在于作用链的起始位置,变量在作用链越深,访问的数据越慢,全局变量位于作用链的最末端.

3.把常用的变量存在局部变量中改善性能.

##DOM编程(重排和重绘)
1.最小化访问DOM,把需要改变的在js中处理完成,一次改变为最优.

2.多次用到DOM节点,存储在变量中在引用.

3.引用HTML集合最好拷贝到一个数组中,由于它连接底层文档,不要直接改变它.

4.离线操作DOM树,使用缓存,减少访问布局的次数

5.动画中绝对定位,使用拖放代理

6.事件委托减少处理器的数量

算法和流程控制

1.改善循环的最佳方式是减少每次迭代的运算量和减少循环迭代次数.

##字符串和正则表达式
1.当连接数量巨大或者尺寸巨大的字符串时,数组合并是唯一在IE及更早版本中性能优化的合理方式.

##快速响应的用户界面
1.任何javaScript任务都不应执行超过100毫秒,过长的时间会导致UI更新出现延迟,产生负面影响.

2.worker在游览器UI线程之外运行代码.

3.可用定时器切割成多个线程.

##Ajax
1.五中常用的数据请求方式:XMLHttpRequest(XHR),动态脚本注入,iframe,comet,multipart XHR

2.XHR发送数据到服务器时,Get方式会更快.Get请求只发送一个数据包,而Post至少发送两个数据包,一个装载头,另一个装载Post正文

3.以即时通信为代表的web应用程序对数据的Low Latency要求，传统的基于轮询的方式已经无法满足，而且也会带来不好的用户体验。于是一种基于http长连接的“服务器推”技术便被hack出来。这种技术被命名为 Comet
